Les commandes de base

  • git init crée un nouveau dépôt Git.
  • git status inspecte le contenu du répertoire de travail et de la zone de transit.
  • git add filename ajoute des fichiers du répertoire de travail à la zone de transit.
  • git add -A ajoute tous les fichiers du répertoire de travail et sous-dossiers à la zone de transit.
  • git add . ajoute tous les fichiers du répertoire de travail à la zone de transit.
  • git diff montre la différence entre le répertoire de travail et la zone de transit.
  • git commit stocke définitivement les changements de fichiers de la zone dans la base de données locale.
  • git log montre une liste de tous les commits précédents.

Comment revenir sur ses pas?

  • HEAD représente le commit le plus récent.
  • git show HEAD montre le commit le plus récent?
  • git checkout HEAD filename supprime les changements dans le répertoire de travail.
  • git reset HEAD filename annule les changements de fichiers dans la zone de transit.
  • git reset commit_SHA permet de revenir à un commit précédent dans votre historique de commit.

Envoyer son dossier sur Githup

  • git remote add origin + adresse du dépot.
  • git branch -M main
  • git push -u origin main

Git Branching

  • git branch liste toutes les branches d'un projet Git et nous indique sur quelle branche on se trouve.
  • git branch new_branch_name crée une nouvelle branche.
  • git checkout branch_name permet de passer d'une branche à une autre.
  • git merge branch_name permet d'inclure tous les changements dans une branche vers la branche master.
  • git branch -d branch_name efface les branches demandées après un git merge.
  • git branch -D branch_name efface les branches demandées même si elle n'ont pas été mergées avec la branche main.

Git Teamwork

  • Remote c'est une dépôt qui permet à plusieurs collaborateurs de travailler ensemble sur un même projet git.
  • git clone remote_location clone_name permet de créer une copie locale d'un dépot.
    • remote_location dit à git où trouver le dépôt.
    • clone_name est le nom que l'on donne au fichier dans lequel Git va cloner le dépot.
  • git remote -v liste les remotes d'un projet.
  • git fetch récupère le travail de la copie distante dans la copie locale.
  • git merge origin/master intègre les changements à notre branche master locale.
  • git push origin your_branch_name va pousser notre branche vers le serveur distant, origin.
